Использование миграций является хорошей практикой изменения структуры базы данных.


Создание миграции:

symfony console make:migration


Был сформирован файл миграции, который содержит все запросы, которые нужно выполнить в базе данных, чтобы отображение приложения соответствовало структуре базы данных. Название для каждой миграции формируется уникальное, пример “migrations/Version20210531205645.php”


Пример миграции:

/*
* Название миграции. Оно будет записано в таблице “doctrine_migration_versions”. 
* Предназначение таблицы: вести учет примененных миграций в базе данных
**/
final class Version20210610094917 extends AbstractMigration
{
    /*
    * Описание миграции
    **/
    public function getDescription() : string
    {
        return '';
    }

    /*
    * При применении миграции, будут выполнены запросы из этого метода
    **/
    public function up(Schema $schema) : void
    {
        $this->addSql('ALTER TABLE product ALTER uuid SET NOT NULL');
    }

    /*
    * При отмене миграции, будут выполнены запросы из этого метода
    **/
    public function down(Schema $schema) : void
    {
        $this->addSql('CREATE SCHEMA public');
        $this->addSql('ALTER TABLE product ALTER uuid DROP NOT NULL');
    }
}


После этого вы можете выполнить миграцию:

symfony console doctrine:migrations:migrate


Узнать статус примененных миграций вы можете командой:

symfony console doctrine:migrations:status


Сформировать список с пустой миграцией:

symfony console doctrine:migrations:generate
