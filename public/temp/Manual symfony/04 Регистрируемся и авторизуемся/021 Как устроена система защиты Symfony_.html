Система защиты в среде symfony предоставляется бандлом symfony/security-bundle.


Настройка аутентификации пользователя:

  1. Создание класса пользователя.

    symfony console make:user (укажем название класса, хранить ли в базе данных, уникальное поле отображения, используется ли хеширование пароля)

  2. Выполнить миграции, которые создадут таблицу с пользователями.

    symfony console make:migration

    symfony console doctrine:migrations:migrate

  3. Добавить в security.yaml раздел с провайдером пользователей “app_user_provider”. Основная задача которого является обновление данных пользователя из сессии.

  4. Добавить в security.yaml раздел с хешированием паролей “password_hashers”, в котором можно указать путь к классу сущности User и алгоритм шифрования.

  5. Добавить в security.yaml в раздел “firewalls” новый фаервол.

  6. Создать аутентификатор для фаервола и указать его у фаервола. Аутентификатор позволяет контролировать весь процесс обработки введенных данных пользователя при попытке авторизоваться.

  7. Создать входной маршрут, на который пользователь может отправлять данные, для попытки авторизоваться.

  8. Проставить проверки на роли пользователя, для доступа к защищенным разделам.


Фаервол в симфони - система аутентификации, в которой указывается область действия, способ авторизации пользователей. На каждый запрос, может распространяться только один фаервол.
